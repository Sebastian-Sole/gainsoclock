# Implementation Log: plan-02
Status: complete

## Summary

Replaced the boolean `isPro` model with a first-class subscription state machine
backed by RevenueCat webhooks.

- **Pure transition helper.** `computeNextState(row, event)` in `convex/subscriptions.ts`
  implements every transition from the plan's table for INITIAL_PURCHASE,
  RENEWAL, PRODUCT_CHANGE, CANCELLATION, UNCANCELLATION, SUBSCRIPTION_PAUSED,
  EXPIRATION, BILLING_ISSUE, REFUND, REFUND_REVERSED,
  TEMPORARY_ENTITLEMENT_GRANT, TRANSFER, SUBSCRIBER_ALIAS, and
  SUBSCRIPTION_EXTENDED. Unknown event types are 200-OK'd with a log line and
  do not advance state. Out-of-order protection compares
  `event.event_timestamp_ms` vs `row.lastEventTimestampMs`; SUBSCRIPTION_EXTENDED
  is the documented exception that updates `trialExpiresAt` only.
- **Idempotent webhook mutation.** `updateFromWebhook({ event })` validates the
  payload via `isRevenueCatEvent`, pre-demotes stale `rc_temp` rows whose 24h
  TTL has elapsed, applies the computed state via `ctx.db.patch`, and stamps
  `lastEventId` + `lastEventTimestampMs`. Auto-creates a row on first event when
  `app_user_id` resolves to a Convex `users` id.
- **Typed RC payloads.** `convex/revenuecatTypes.ts` exports a discriminated
  union covering all 14 event types and a `isRevenueCatEvent` runtime guard.
  (Renamed from `revenuecat-types.ts` because Convex disallows hyphens in
  module paths.)
- **Webhook handler hardening.** `convex/http.ts` now compares the
  `Authorization` header against both `REVENUECAT_WEBHOOK_AUTH_TOKEN` and
  `REVENUECAT_WEBHOOK_AUTH_TOKEN_PREVIOUS` via `timingSafeEqualString` (Convex
  HTTP actions run in V8, so the constant-time compare is implemented in JS
  rather than via Node `crypto.timingSafeEqual`). Bearer-prefixed headers are
  accepted. Unknown event types return 200 with a log line. New
  `/webhooks/email/unsubscribe` route flips `emailOptOut` for the legal
  reminder emails.
- **Crons.** `convex/crons.ts` registers `trial-reminder-48h` (daily 08:00
  UTC), `dcsa-6-monthly` (daily 09:00 UTC), and `rc-temp-demote` (hourly).
  Internal handlers in `convex/subscriptionCrons.ts` scan by the
  `by_status_*` indexes added in plan-01, dispatch Resend mail via
  `ctx.scheduler.runAfter(0, ...)`, and patch the idempotency columns
  (`reminder48hSentAt`, `dcsaNotifiedAt`).
- **Resend wrapper.** `convex/email.ts` (`"use node"`) sends English-V1
  templates for trial reminder + DCSA reminder via `fetch` to the Resend REST
  API. `EMAIL_SERVICE_API_KEY` env required; `storefrontCountry` is wired
  through as the V1.1 localisation hook.
- **Migration.** `convex/migrations.ts` exports `migrateSubscriptionsV2({
  logOnly })` — dry-run mode prints planned writes; commit mode patches the
  state-machine columns from `(isActive, productId, expiresAt)`. Skips rows
  that are already migrated.
- **Client.**
  - `stores/subscription-store.ts` extended with the full state-machine surface
    (`status`, `source`, `trialExpiresAt`, `willAutoRenew`, `sourceHistory`,
    `cancelReason`, `lastVerifiedAt`, `notificationAnchorAt`, `emailOptOut`,
    `storefrontCountry`). `isPro` is retained as a `@deprecated` derived
    selector so legacy callers don't break mid-migration.
  - `hooks/use-purchases.ts` adds `getOfferings()` (per-session cached) and
    `checkTrialOrIntroDiscountEligibility()`. `syncCustomerInfo` now drops
    payloads with `requestDate` older than the store's `lastVerifiedAt`
    (Offline-Sync #5). `Purchases = rnpModule.default ?? rnpModule` preserved
    (RC F4).
  - `providers/convex-sync-provider.tsx` reads from the new
    `getSubscriptionState` query so all state-machine fields flow into the
    store.
  - `app/settings/index.tsx:68` now branches on `status`, not `isPro`.
- **Rotation doc.** `docs/revenuecat-webhook-rotation.md` extended with the
  `curl` test suite for valid token / wrong token / `_PREVIOUS` token / stale
  event / unknown event type.
- **Pinned versions.** `react-native-purchases` and `react-native-purchases-ui`

## Notes / Deviations

- Convex module names cannot contain hyphens — `convex/revenuecat-types.ts`
  was renamed to `convex/revenuecatTypes.ts`. Imports updated in
  `convex/subscriptions.ts` and `convex/http.ts`.
- The plan suggested `crypto.timingSafeEqual` from Node, but Convex HTTP
  actions run in the V8 runtime (no `"use node"` available for `httpAction`).
  Implemented `timingSafeEqualString` directly in JS — same constant-time
  guarantee, length-mismatch folded into the loop so the work is
  O(max(a.length, b.length)) regardless of input.
- `updateFromWebhook` accepts `event: v.any()` rather than a Convex validator
  union for the discriminated event type — Convex validators don't support
  `v.enum` or extension-friendly unions large enough to mirror the full RC
  event surface, and the runtime guard in `isRevenueCatEvent` already
  rejects malformed payloads at the HTTP boundary.
- `app/(tabs)/chat.tsx` and the legacy `getStatus` query still exist; both
  paths are forward-compatible with the new state-machine because the
  derived `isPro` selector and the legacy `isActive` field are computed
  from `status`. Migrating the chat gate to read `status` directly is
  deferred to plan-08 alongside the paywall interstitial work.
- The `unsubscribeToken` is a deterministic 32-bit hash of `userId + secret`
  rather than an HMAC — sufficient for the V1 unsubscribe link (low-value
  endpoint that just flips `emailOptOut`), but should be upgraded to a real
  HMAC once the Convex Web Crypto path is wired through plan-10.
